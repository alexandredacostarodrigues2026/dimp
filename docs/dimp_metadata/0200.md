# Registro 0200 - Meio de Captura

## Nivel hierarquico

- Nivel: `1`
- Pai: `0000`
- Referenciado por: `1110` (FK cadastral — COD_MCAPT)

## Funcao

Cadastra o terminal/tecnologia usada nas operacoes. O `COD_MCAPT` e a chave de
ligacao com as operacoes diarias: todo `1110` deve ter um `0200` correspondente
no mesmo arquivo (regra de residuo proibido, DIMP V10).

## Campos (layout DIMP V10)

| Posicao | Campo | Tipo | Descricao |
|---|---|---|---|
| 0 | REG | X | Texto fixo `0200` |
| 1 | COD_MCAPT | X | Codigo do meio de captura (livre atribuicao, unico por arquivo) |
| 2 | COD_IP | X | CNPJ da IP responsavel pelo meio de captura |
| 3 | TIPO_TECNOLOGIA | N | Tipo de tecnologia — ver tabela abaixo |
| 4 | FLAG | X | Indicador interno |
| 5 | MARCA | X | Marca/descricao do equipamento |

## Tipos de tecnologia (TIPO_TECNOLOGIA)

| Codigo | Descricao | Isento do limiar CPF |
|---|---|---|
| 1 | POS (Point of Sale) | Sim |
| 2 | Mobile | Sim |
| 3 | E-commerce | Sim |
| 4 | TEF (Transferencia Eletronica de Fundos) | Sim |
| 5 | ATM | Nao |
| 6 | Outros | Nao |
| 7 | Pagamento em dinheiro ou outra estrutura | N/A |

## Ligacao cadastral

```
0200.COD_MCAPT  <---->  1110.COD_MCAPT
```

JOIN no banco:
```sql
SELECT r2.cod_mcapt, r2.tipo_tecnologia, r11.dt_operacao, r11.valor_total
FROM reg_0200 r2
JOIN lote l ON l.cnpj_ip = r2.cnpj_ip
JOIN reg_1110 r11 ON r11.chave_lote = l.chave_lote AND r11.cod_mcapt = r2.cod_mcapt
```

## Tabela SQLite: reg_0200

| Coluna | Tipo | Descricao |
|---|---|---|
| chave_0200 | TEXT PK | chave_lote\|cod_mcapt |
| chave_lote | TEXT FK→lote | Lote de transmissao |
| cnpj_ip | TEXT | CNPJ da IP declarante |
| cod_mcapt | TEXT | Codigo do meio de captura |
| cod_ip | TEXT | CNPJ do responsavel pelo terminal |
| tipo_tecnologia | TEXT | Codigo da tecnologia (1-7) |
| marca | TEXT | Descricao/marca |

Indice: `idx_0200_mcapt (cnpj_ip, cod_mcapt)`

## Regras de negocio

- Todo `1110.COD_MCAPT` deve ter correspondente em `0200` do mesmo arquivo.
  Violacao = residuo proibido pelo V10; parser emite WARNING e app exibe na
  secao "Validacao de Cadastro vs Operacoes".
- `COD_MCAPT` deve ser unico por arquivo (nao pode repetir dentro do mesmo 0000).
- Tecnologias 1-4 (POS, Mobile, E-commerce, TEF) isentam do limiar de CPF no `1100`.

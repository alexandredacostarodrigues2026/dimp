# Registro 1100 - Resumo Mensal

## Nivel hierarquico

- Nivel: `2`
- Pai: `0000`
- Referencia cadastral: `0100` (FK por COD_CLIENTE)
- Filho: `1110`

## Funcao

Consolida o movimento mensal de um cliente no periodo declarado (`DT_INI` a
`DT_FIN`). Cada `1100` abre um contexto; os `1110` seguintes pertencem a ele
ate o proximo `1100` ou fim do arquivo.

## Campos (layout DIMP V10)

| Posicao | Campo | Tipo | Descricao |
|---|---|---|---|
| 0 | REG | X | Texto fixo `1100` |
| 1 | COD_IP_PAR | X | CNPJ da IP parceira (liquidante) |
| 2 | COD_CLIENTE | X | Chave para `0100.COD_CLIENTE` |
| 3 | IND_COMEX | N | Indicador de comercio exterior |
| 4 | IND_EXTEMP | N | 0=normal do periodo, 1=extemporaneo |
| 5 | DT_INI | N | Data inicial do periodo (AAAAMMDD) |
| 6 | DT_FIN | N | Data final do periodo (AAAAMMDD) |
| 7 | VALOR | N | Valor total do periodo |
| 8 | QTD | N | Quantidade de transacoes do periodo |

## Chave composta no banco

```
chave_1100 = cnpj_ip|dt_tx|hora_tx|cod_cliente|dt_ini|dt_fin
chave_pai_0100 = cnpj_ip|dt_tx|hora_tx|cod_cliente
```

## Chave de periodo para retificacao

A identificacao do periodo usa `DT_INI + DT_FIN` (nao `competencia`), cobrindo
declaracoes extemporaneas onde o mes dos dados difere do mes de envio.

## Regra de soma

```text
1100.VALOR == SUM(1110.VALOR_TOTAL_DIARIO)
```

Divergencias sao exibidas na secao "Comparacao de Valores" do app.

## Regras de negocio

- Ao ler novo `1100`, o `1100` anterior e fechado e validado.
- `COD_CLIENTE` deve existir em `0100` — violacao = residuo proibido V10.
- `IND_EXTEMP=1` e proibido em retificacoes (finalidade=2).
- Retificacao (finalidade=2): DELETE WHERE cnpj_ip + dt_ini + dt_fin + ind_extemp='0',
  preservando registros extemporaneos de periodos anteriores.

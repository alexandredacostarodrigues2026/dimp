# Registro 0100 - Cadastro de Cliente

## Nivel hierarquico

- Nivel: `1`
- Pai: `0000`
- Referenciado por: `1100` (FK cadastral — COD_CLIENTE)

## Funcao

Cadastra o cliente/estabelecimento informado no DIMP. O `COD_CLIENTE` e a chave
de ligacao com o bloco de operacoes: todo `1100` deve ter um `0100` correspondente
no mesmo arquivo (regra de residuo proibido, DIMP V10).

## Campos (layout DIMP V10)

| Posicao | Campo | Tipo | Descricao |
|---|---|---|---|
| 0 | REG | X | Texto fixo `0100` |
| 1 | COD_CLIENTE | X | Chave tecnica do cliente (livre atribuicao da IP) |
| 2 | CNPJ | N | CNPJ quando cliente pessoa juridica |
| 3 | CPF | N | CPF quando cliente pessoa fisica |
| 4 | NOME_RAZAO_SOCIAL | X | Nome ou razao social |
| 5 | LOGRADOURO | X | Endereco |
| 6 | CEP | N | CEP |
| 7 | COD_MUNICIPIO | N | Codigo IBGE do municipio |
| 8 | UF | X | UF do estabelecimento |
| 9 | NOME_CONTATO | X | Nome do responsavel |
| 10 | TELEFONE | X | Telefone de contato |
| 11 | EMAIL | X | E-mail de contato |
| 12 | DT_INICIO | N | Data de inicio das operacoes (AAAAMMDD) |
| 13 | FLAG | X | Indicador de registro (uso interno IP) |

## Ligacao cadastral

```
0100.COD_CLIENTE  <---->  1100.COD_CLIENTE
```

JOIN no banco:
```sql
SELECT r0.nome_razao_social, r1.valor
FROM reg_0100 r0
JOIN reg_1100 r1 ON r0.cnpj_ip = r1.cnpj_ip AND r0.cod_cliente = r1.cod_cliente
```

## Tabela SQLite: reg_0100

| Coluna | Tipo | Descricao |
|---|---|---|
| chave_0100 | TEXT PK | chave_lote\|cod_cliente |
| chave_lote | TEXT FK→lote | Lote de transmissao |
| cnpj_ip | TEXT | CNPJ da IP declarante |
| cod_cliente | TEXT | Codigo do cliente |
| cnpj | TEXT | CNPJ do cliente (se PJ) |
| cpf | TEXT | CPF do cliente (se PF) |
| nome_razao_social | TEXT | Nome ou razao social |
| uf | TEXT | UF do estabelecimento |

Indice: `idx_0100_cliente (cnpj_ip, cod_cliente)`

## Regras de negocio

- Todo `1100.COD_CLIENTE` deve ter correspondente em `0100` do mesmo arquivo.
  Violacao = residuo proibido pelo V10; parser emite WARNING e app exibe na
  secao "Validacao de Cadastro vs Operacoes".
- CPF deve ser avaliado com o limiar: valor >= R$ 3.375,00 ou qtd >= 30
  transacoes exige identificacao do CPF (regra de limiar DIMP V10).
- `COD_CLIENTE` e de livre atribuicao da IP; nao e CPF nem CNPJ diretamente.

# Registro 0000 - Abertura

## Nivel hierarquico

- Nivel: `0`
- Pai: nenhum
- Filhos relacionados neste contexto: `0100`, `0200`, `1100`

## Funcao

Abre o arquivo DIMP e identifica a instituicao de pagamento declarante, a UF e
o periodo informado.

## Campos chave (layout oficial V10)

| Posicao no parser | Campo | Tipo | Uso |
| --- | --- | --- | --- |
| `0` | `REG` | X | Deve ser `0000` |
| `1` | `VERSAO` | X | Versao do leiaute |
| `2` | `FINALIDADE` | N | Finalidade da entrega |
| `3` | `UF` | X | Unidade federada |
| `4` | `CNPJ_IP` | N | CNPJ da instituicao declarante |
| `5` | `NOME_IP` | X | Nome empresarial da instituicao |
| `6` | `DT_INI` | N | Data inicial no formato `AAAAMMDD` |
| `7` | `DT_FIN` | N | Data final no formato `AAAAMMDD` |
| `8` | `SITUACAO` | N | Situacao da declaracao |
| `9` | `COMPETENCIA` | N | Mes de validacao no formato `AAAAMM` |

## Campos de auditoria interna (nao oficiais — nao constam no layout V10)

Extraidos do cabecalho tecnico da primeira linha do arquivo (envelope de protocolo),
que antecede o primeiro registro pipe-delimitado. Nunca devem ser adicionados ao
arquivo `.txt` submetido ao validador oficial.

| Campo no dataclass | Formato | Exemplo | Origem |
| --- | --- | --- | --- |
| `dt_tx` | `AAAAMMDD` | `20260317` | Regex `\d{4}/\d{2}/\d{2}` no cabecalho |
| `hora_tx` | `HHMMSS` | `183025` | Regex `\d{2}:\d{2}:\d{2}` no cabecalho |

## Regras para IA

- Usar `CNPJ_IP` como identificador da operadora/instituicao declarante.
- O registro inicia o contexto global do arquivo.
- `dt_tx` e `hora_tx` identificam o momento exato da transmissao e sao uteis
  para auditoria e joins entre arquivos de diferentes competencias.
- Linhas antes do primeiro `|0000|` sao envelope tecnico e sao ignoradas pelo
  parser DIMP (exceto para extracao de `dt_tx`/`hora_tx`).

# Registro 0000 - Abertura

## Nivel hierarquico

- Nivel: `0`
- Pai: `00000` (via `chave_pai_00000`)
- Filhos relacionados neste contexto: `0100`, `0200`, `1100`

## Funcao

Abre o arquivo DIMP e identifica a instituicao de pagamento declarante, a UF e
o periodo informado.

## Campos chave (layout oficial V10)

| Posicao no parser | Campo | Tipo | Uso |
| --- | --- | --- | --- |
| `0` | `REG` | X | Deve ser `0000` |
| `1` | `VERSAO` | X | Versao do leiaute |
| `2` | `FINALIDADE` | N | Finalidade da entrega |
| `3` | `UF` | X | Unidade federada |
| `4` | `CNPJ_IP` | N | CNPJ da instituicao declarante |
| `5` | `NOME_IP` | X | Nome empresarial da instituicao |
| `6` | `DT_INI` | N | Data inicial no formato `AAAAMMDD` |
| `7` | `DT_FIN` | N | Data final no formato `AAAAMMDD` |
| `8` | `SITUACAO` | N | Situacao da declaracao |
| `9` | `COMPETENCIA` | N | Mes de validacao no formato `AAAAMM` |

## Chaves de ligacao (campos calculados — nao oficiais)

| Campo | Valor | Direcao |
| --- | --- | --- |
| `chave_pai_00000` | `dt_tx\|hora_tx` do registro `00000` pai | FK para cima |
| `chave_0000` | `cnpj_ip` (campo 04 do layout) | PK deste registro |

`chave_0000` e propagada para todos os registros filhos (`0100`, `0200`, `1100`, `1110`, `1115`)
como `chave_pai_0000`, garantindo que qualquer registro do arquivo possa ser vinculado ao CNPJ
da instituicao declarante sem joins intermediarios.

Exemplo: `chave_0000 = "22896431000382"` (PicPay), `"03130170000189"` (Brasil Card).

## Regras para IA

- `chave_0000 = cnpj_ip` e o identificador unico da entidade declarante dentro da aplicacao.
- Todo registro do arquivo herda `chave_pai_0000 = cnpj_ip` como indice de busca.
- O registro inicia o contexto global do arquivo; todos os filhos pertencem a este CNPJ.
- Linhas antes do primeiro `|0000|` sao envelope tecnico ignorado pelo parser
  (exceto para extracao de `dt_tx`/`hora_tx` via `Registro00000`).

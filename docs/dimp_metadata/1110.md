# Registro 1110 - Operacao Diaria

## Nivel hierarquico

- Nivel: `3`
- Pai: `1100`
- Referencia cadastral: `0200` (FK por COD_MCAPT)
- Filho: `1115`

## Funcao

Consolida as transacoes de um dia para um determinado meio de captura dentro
do resumo mensal `1100`. Um `1110` por combinacao (dia + cod_mcapt + liquidante).

## Campos (layout DIMP V10)

| Posicao | Campo | Tipo | Descricao |
|---|---|---|---|
| 0 | REG | X | Texto fixo `1110` |
| 1 | COD_MCAPT | X | Chave para `0200.COD_MCAPT` |
| 2 | DT_OPERACAO | N | Data da operacao (AAAAMMDD) |
| 3 | VALOR_TOTAL_DIARIO | N | Valor total do dia |
| 4 | QTD_TOTAL | N | Quantidade de transacoes do dia |
| 5 | CNPJ_IP | X | CNPJ da IP liquidante |

## Chave composta no banco

```
chave_1110 = cnpj_ip|dt_tx|hora_tx|cod_cliente|dt_ini|dt_fin|cod_mcapt|dt_operacao
chave_pai_0200 = cnpj_ip|dt_tx|hora_tx|cod_mcapt
```

## Regra de soma

```text
1110.VALOR_TOTAL_DIARIO == SUM(1115.VALOR_TRANSACAO)
```

Divergencias sao exibidas na secao "Comparacao de Valores" do app.

## Regras de negocio

- Ao ler novo `1110`, o `1110` anterior e fechado e validado.
- `COD_MCAPT` deve existir em `0200` — violacao = residuo proibido V10.
- Todo `1110` pertence ao `1100` ativo; `1110` sem `1100` e ignorado com WARNING.
- Tecnologia do meio de captura (via `0200`) influencia o calculo do limiar de
  CPF no `1100` pai: tecnologias 1-4 isentam do limiar.
